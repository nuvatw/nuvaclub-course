# 1.0 本章概要（內部用）

## 1.0.1 章節目標

這章的教學策略是：用「狗狗 GPT（只回汪）」當最小可控實驗，把 Instruction、Preview、測試、Publish/Share 的核心流程練熟；再把同一套流程套到「GPT A（Productivity assistant）」。

能在 GPT Builder 內完成一個最小可用 GPT（MVP），並用「可測試輸出格式」鎖定行為。

- **課堂 MVP**：Dog GPT（輸出格式 = 回覆必須完全等於「汪」）
- **核心能力**：把「規格」寫成可以被驗收的「配置」，並在 Preview 當場測通。
- **建立第一個作品**：GPT A（Productivity assistant）v0（課後套用）
  - 課堂先不做完整 GPT A：先用 Dog GPT 把流程跑通、把「鎖行為」肌肉練起來。
  - 課後作業：把同樣流程（需求→bullets→Instruction→Preview→tests→Publish Private）套到 GPT A v0。
- **建立「資料/隱私」基線**：知道什麼能放、什麼不能放、以及外部 API 的資料外流風險
  - 本章在 1.6 用 Share 設定先做第一道風險閘門（先 Only me/Invite-only）。
  - 明確提醒：創作者看不到使用者與該 GPT 的對話（所以不是「我可以偷看」，但也不代表你可以亂放機密）。
  - 外部 API/Apps 的風險基線：使用外部服務時，相關輸入可能被送到第三方，且 OpenAI 不審核/不控制第三方如何使用或儲存資料（只用你信任的 API/服務）。
- **建立「測試先行」習慣**：每次改 Instructions 都要跑一組固定測試題（regression）
  - 本章先做最小 tests.md（一般/極端/Prompt Injection 三類）
  - 強制養成「Bug → Fix → Retest」紀錄，為後續擴充 regression bank 打底。

## 1.0.2 教學策略

**為什麼 Chapter 1 用 Dog GPT，而不是直接做 GPT A？**

核心理由：**降低變因、放大因果。**

- Dog GPT 的需求極簡（任何輸入→只回「汪」），可以讓學員「只專注一件事」：Instruction 是否真的能鎖行為。
- 一旦學員看懂「行為鎖定 = 可測試」，後面把需求換成 GPT A（會議整理/待辦/計畫書框架）只是規格變長，流程不變。

**本章刻意不教（避免失焦）**

- Chapter 2 才教：Instruction 進階寫法（behavior contract、few-shot、可維護性）
- Chapter 3 才教：Conversation starters、Knowledge（RAG）、Capabilities（Web Search / Canvas / Image Generation / Code Interpreter）
- Version history（回到舊版本）在 Chapter 2 教（本章只先點到）。

## 1.0.4 核心概念

- **Instruction Engineering**：把需求寫成可執行的指示（不是許願）。
- **Behavior contract（行為規格）**：用「必須/不得/遇到 X 就 Y」寫出可驗收的規則。
- **可測試輸出格式**：輸出要能被客觀判定 Pass/Fail（Dog GPT 用「完全等於汪」示範）。
- **Failure modes（失敗模式）**：格式漂移、被越獄、亂編、越權。
- **Prompt Injection（越獄/指令複寫）**：試圖叫模型忽略規則；本章在 tests.md 必測。
- **Sharing levels（分享層級）**：Only me/Invite-only → Workspace → Link → Public（影響面=風險面）。
- **隱私基線**：創作者看不到使用者對話；外部 API 可能把輸入送到第三方。

## 1.0.5 常見 Q&A

**Q1：誰可以「建立/編輯」GPT？Free 可以嗎？**
A：建立 GPT 的 Builder 功能目前屬於付費方案（如 Pro/Plus/Team/Enterprise/Edu）；Free 使用者可以使用 GPTs，但通常不能建立或發布。

**Q2：我 GPT 不能生成檔案/圖表/分析，為什麼？**
A：先檢查是否啟用 Code Interpreter & Data Analysis（能力沒開就容易做不起來）。

**Q3：Knowledge 最多可以放幾個檔案？**
A：Knowledge 可附加最多 20 個檔案；也有單檔大小與 token 上限等限制（後面章節再教怎麼用）。

**Q4：創作者能看到使用者跟 GPT 的對話嗎？**
A：官方說明是創作者無法存取使用者與該 GPT 的對話內容。

**Q5：外部 API/Apps 的資料風險是什麼？**
A：只要 GPT 會呼叫外部服務，你輸入的相關內容可能被送到第三方，且 OpenAI 不控制第三方如何使用/儲存資料；只用你信任的服務。

**Q6：分享層級有哪些？我第一版應該選哪個？**
A：分享層級可包含 Invite-only/Workspace/（有連結）/Public；第一版建議 Only me / Invite-only，等測試通過再擴散。

**Q7：我可以回到舊版本嗎？**
A：編輯 GPT 時可看 Version History 並可 revert；本課只先知道「有得回」，第 2 章才教怎麼用它做安全迭代。
