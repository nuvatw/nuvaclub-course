# 1.2 講 Metadata

## 教學目標

- 學員認得 Builder 的欄位：哪些是門面、哪些決定行為（image / name / description / instruction）
- 先講「後面章節會教什麼」，避免今天課堂發散
- 這章只做「一段話狗狗」

---

> [!SCRIPT]
> 我們現在進 Builder。你先記一個分法：
> Name / Description / Icon 是門面：讓人知道它是誰、做什麼。
> Instructions 是靈魂：決定它怎麼工作、能做什麼、不能做什麼。
> 你可能還會看到其他東西：
> Conversation starters、Knowledge、Capabilities（像 Web Search、Canvas、Image Generation、Code Interpreter & Data Analysis）
> 但我先幫你把期待收斂：
> Chapter 1 我們只用 Instructions 做最小行為鎖定。
> Chapter 2 才會教怎麼把 Instruction 寫得更像工程規格；
> Chapter 3 才會教 starters、knowledge、capabilities 怎麼搭配。
> 所以今天：你只要跟著我填幾個欄位，做出一隻只會「汪」的狗就好。

---

## 螢幕操作腳本

> 介面可能會隨版本略有不同，但核心欄位一定存在。你照「欄位名稱」找即可。

1. 打開 GPT Builder：`https://chatgpt.com/gpts/editor`
2. 指給大家看：通常會有 Create / Configure 分頁
3. 點 **Configure**（今天主要在這裡）
4. 依序填：
   - **Name**：狗狗 GPT
   - **Description**：不管你說什麼我都只回「汪」
   - **Icon / image**：先略過（你可以說「有時間再做」）
5. 找到 **Instructions** 欄位：先不要貼，告訴大家「下一節會貼模板」
6. 快速滑過（只指一下，不教）：
   - Conversation starters
   - Knowledge
   - Capabilities（讓大家知道位置即可）

## Builder 欄位分類

| 欄位 | 分類 | 本章是否教 |
|------|------|-----------|
| Name | 門面 | 填 |
| Description | 門面 | 填 |
| Icon / Image | 門面 | 跳過 |
| **Instructions** | **靈魂** | **本章重點** |
| Conversation starters | 產品化 | Ch4 |
| Knowledge | RAG | Ch3 |
| Capabilities | 工具 | Ch4 |
| Actions | API | Ch5 |
